## ISSUE-051: Resumo IA historico antigo com truncamento "hoes" no valor

**Persona:** Priya (Edge Cases)
**Severidade:** P3
**Area:** Historico — dados historicos
**Data:** 2026-03-30 10:20
**Status:** ABERTO

### Classificacao de Severidade
P3 — Afeta apenas dados historicos anteriores ao fix. Novas buscas geram valores corretos. Nao e retroativo.

### Passos para Reproduzir
1. Navegar para https://smartlic.tech/historico
2. Localizar busca de Engenharia SP de 29/03/2026, 20:50
3. Ler resumo IA

### Comportamento Observado
Resumo mostra "...com valores totais que ultrapassam R$ 374.496.875,20hões." — sufixo "hoes" truncado do LLM.

### Comportamento Esperado
Valor deveria ser "R$ 374.496.875,20" ou "R$ 374,5 milhoes" — sem truncamento.

### Notas Tecnicas
- Busca antiga (pre-fix). O post-processing regex do commit `b1073430` trata "R$ X mil" mas nao o padrao "R$ X,YYhoes" (truncamento de "milhoes").
- Dados historicos nao sao retroativos — afeta apenas resumos gerados antes do fix.
- Fix: regex adicional para detectar e limpar sufixos truncados como "hoes", "lhoes" apos valores monetarios.

### Fix Sugerido
Adicionar regex no post-processing de `llm.py`: `r'(R\$\s*[\d.,]+)\d*h[oõ]es'` → `\1`

**Eventos FALTANDO (4):**
| Evento | Impacto | Handler no Codigo |
|--------|---------|-------------------|
| `checkout.session.async_payment_succeeded` | **Boleto pago NUNCA ativa assinatura!** | `handle_async_payment_succeeded()` ✅ |
| `checkout.session.async_payment_failed` | Boleto expirado nao notifica usuario | `handle_async_payment_failed()` ✅ |

**Fix:** Stripe Dashboard → Developers → Webhooks → `we_1SzSa59FhmvPslGYw2TB83Jw` → Edit → Adicionar os 4 eventos. A restricted key do CLI nao tem permissao para alterar.

**Risco:** Cliente paga via Boleto → checkout cria subscription `pending_payment` → Stripe confirma pagamento → evento `async_payment_succeeded` enviado → Stripe recebe 404 (evento nao registrado) → assinatura fica em `pending_payment` PARA SEMPRE → cliente paga mas nao recebe acesso.
