## ISSUE-053: Admin sub-pages (cache/metrics/SLO) bloqueadas para admin

**Persona:** Jordan (Security)
**Severidade:** P2
**Area:** Admin
**Data:** 2026-03-30 14:05
**Status:** ABERTO

### Classificacao de Severidade
P2 — Links existem na pagina admin principal mas as sub-pages retornam "Acesso Restrito" mesmo para o usuario admin (tiago.sasaki@gmail.com, plano SmartLic Pro).

### Passos para Reproduzir
1. Login como tiago.sasaki@gmail.com (admin)
2. Navegar para https://smartlic.tech/admin — funciona OK (7 usuarios listados)
3. Clicar em "Cache" (https://smartlic.tech/admin/cache) — "Acesso Restrito"
4. Clicar em "Metrics" (https://smartlic.tech/admin/metrics) — "Acesso Restrito"
5. Clicar em "SLOs" (https://smartlic.tech/admin/slo) — "Acesso Restrito"

### Comportamento Observado
Todas as 3 sub-pages mostram "Esta pagina e exclusiva para administradores" com link "Voltar" para /buscar.

### Comportamento Esperado
Admin deveria ter acesso a todas as sub-pages de admin (cache, metrics, SLO).

### Notas Tecnicas
- A pagina principal /admin funciona e mostra dados (7 usuarios, uptime, fontes ativas)
- Possivel causa: sub-pages verificam `is_master` ao inves de `is_admin`, ou verificam um campo diferente na tabela profiles

- Console: 0 errors

### Fix Sugerido
Verificar o middleware de autorizacao nas paginas admin/cache, admin/metrics, admin/slo — provavelmente checam `is_master` ou um campo ausente. Alinhar com a mesma logica de /admin principal.

## ISSUE-054: Busca Saude timeout com 3 UFs (SP/MG/RJ)

**Persona:** Marcus (Performance) / ICP-04 Mariana
**Severidade:** P2
**Area:** Buscar — timeout
**Data:** 2026-03-30 14:35
**Status:** ABERTO

### Classificacao de Severidade
P2 — Busca falha mas UI oferece fallback inteligente ("Buscar apenas SP e MG") que funciona. Timeout ~85s.

### Passos para Reproduzir
1. Login como admin
2. Buscar: setor=Saude, UFs=SP+MG+RJ, modalidade=Pregao Eletronico, valor R$50k-R$3M, modo=Abertas
3. Aguardar ~85 segundos

### Comportamento Observado
Mensagem: "A analise excedeu o tempo limite. Algumas fontes podem estar lentas."
Sugestao oferecida: "Buscar apenas SP e MG" (funciona e retorna 316 resultados)
Console: [CRIT-070] Client timeout triggered, [CRIT-SSE-FIX] SSE terminal timeout (5s)

### Comportamento Esperado
Busca deveria completar com 3 UFs ou retornar resultados parciais automaticamente.

### Notas Tecnicas
- Saude com 3 UFs grandes (SP+MG+RJ) gera ~1009 oportunidades para analise — volume muito alto
- O fallback "buscar menos UFs" e UX excelente — reduz friccao significativamente
- Com 2 UFs (SP+MG) a busca completou em ~40s com 316 resultados (51 alta, 31 media, 234 baixa)
- Issue inerente ao volume de dados no setor Saude (muito mais editais que outros setores)

### Fix Sugerido (opcional)
Considerar timeout adaptativo por setor — setores com alto volume (Saude, Engenharia) poderiam ter timeout maior ou fazer batching mais agressivo.

## ISSUE-055: Saude — 234 de 316 resultados com relevancia "baixa"

**Persona:** ICP-04 Mariana
**Severidade:** P3
**Area:** Buscar — qualidade de classificacao
**Data:** 2026-03-30 14:45
**Status:** ABERTO

## ICP Findings — Acoes Recomendadas

### CRITICO: Engenharia confunde materiais com obras
O setor "Engenharia, Projetos e Obras" captura massivamente "aquisicao de materiais de construcao" por keyword match. Uma construtora que busca OBRAS (pavimentacao, reforma, construcao) recebe 90%+ de editais de COMPRA DE MATERIAIS. Isso e um problema de product-market fit para o ICP principal (construtoras).

**Acao recomendada:** Considerar sub-classificacao ou flag "tipo de edital" (servicos/obras vs aquisicao/materiais) na pipeline de classificacao. Alternativamente, permitir que o usuario refine com termos exclusivos ("-materiais" ou "somente servicos").

### BOM: Software tem excelente fit
A classificacao para Software e Sistemas e precisa e acionavel. Persona Juliana encontrou 13/25 oportunidades reais. Este e o setor showcase.

### A MELHORAR: Saude precisa de sub-categorias
316 resultados com 234 "baixa" relevancia e muito ruido. Sub-filtros (material hospitalar, medicamentos, odontologia, servicos medicos) resolveriam.
