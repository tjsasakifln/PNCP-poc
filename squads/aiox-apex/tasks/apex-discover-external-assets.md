# Task: apex-discover-external-assets

```yaml
id: apex-discover-external-assets
version: "1.0.0"
title: "External Assets Discovery"
description: >
  Scans the project for assets sourced from external origins (stock photos,
  curated icons, 3D models, web-extracted images). Verifies link integrity,
  license compliance, optimization status, and usage. Answers "which external
  assets are in my project, are they healthy, and am I using them all?"
elicit: false
owner: apex-lead
executor: web-intel
dependencies:
  - tasks/apex-scan.md
  - tasks/web-asset-hunt.md
  - tasks/web-asset-optimize.md
  - checklists/asset-licensing.md
outputs:
  - External asset inventory (source, format, size, license, usage)
  - Broken link report (unreachable URLs, 404s)
  - License compliance status (missing, unknown, restricted)
  - Optimization gaps (unoptimized formats, missing srcset, no lazy-load)
  - Orphan assets (imported but never rendered)
  - External asset health score (0-100)
```

---

## Command

### `*discover-external-assets`

Scans the project for externally-sourced assets. Runs as part of `*apex-audit` or independently.

---

## Discovery Phases

### Phase 1: Identify External Assets

```yaml
asset_detection:
  image_sources:
    scan:
      - "src/**/*.{tsx,jsx,ts,js,css}"
      - "public/**/*.{png,jpg,jpeg,webp,avif,svg,gif}"
      - "app/**/*.{tsx,jsx,ts,js}"
    patterns:
      urls: "https?://.*\\.(png|jpg|jpeg|webp|avif|svg|gif|glb|gltf)"
      imports: "import.*from.*\\.(png|jpg|jpeg|webp|avif|svg|gif)"
      css_urls: "url\\(.*\\.(png|jpg|jpeg|webp|avif|svg|gif)\\)"
      public_assets: "All files in public/ not generated by build"

  stock_photo_indicators:
    - "unsplash.com"
    - "pexels.com"
    - "pixabay.com"
    - "images.unsplash.com"
    - "stock" in filename or path
    - "hero-", "banner-", "bg-" prefixes

  icon_sources:
    - "lucide-react"
    - "heroicons"
    - "phosphor"
    - "@radix-ui/react-icons"
    - "Custom SVGs in src/assets/ or public/icons/"

  model_3d_sources:
    - "*.glb, *.gltf, *.obj, *.fbx files"
    - "sketchfab.com references"
    - "poly.pizza references"
    - "polyhaven.com references"
```

### Phase 2: Verify Link Integrity

```yaml
link_checks:
  per_asset:
    url: "{source URL}"
    status: "reachable | 404 | timeout | redirect | local"
    response_time: "{ms}"
    final_url: "{after redirects}"

  checks:
    broken_links:
      description: "External URLs that return 4xx/5xx"
      severity: HIGH
      action: "Replace with local copy or find alternative"

    slow_assets:
      description: "Assets loading > 2000ms"
      severity: MEDIUM
      action: "Download locally, serve from CDN, or optimize"

    http_not_https:
      description: "Assets served over HTTP (not HTTPS)"
      severity: HIGH
      action: "Update to HTTPS or download locally"

    redirect_chains:
      description: "Assets with 2+ redirects"
      severity: LOW
      action: "Update to final URL"
```

### Phase 3: License Audit

```yaml
license_verification:
  per_asset:
    source: "{origin URL or service}"
    license_type: "CC0 | Unsplash | Pexels | MIT | CC-BY | CC-NC | Unknown"
    commercial_use: true | false | "verify"
    attribution_required: true | false
    attribution_present: true | false

  checks:
    missing_license:
      description: "Asset with no license information"
      severity: HIGH
      action: "Trace origin and verify license before shipping"

    non_commercial_in_commercial:
      description: "CC-NC asset in commercial project"
      severity: CRITICAL
      action: "Replace immediately — legal risk"

    attribution_missing:
      description: "CC-BY asset without attribution in credits"
      severity: MEDIUM
      action: "Add attribution to credits page or README"

    unknown_origin:
      description: "Asset with no traceable source"
      severity: HIGH
      action: "Verify origin or replace with known-licensed alternative"
```

### Phase 4: Optimization Status

```yaml
optimization_checks:
  per_asset:
    current_format: "{png|jpg|svg|webp|avif|glb}"
    optimal_format: "{webp|avif|svg|glb}"
    current_size: "{KB}"
    estimated_optimal_size: "{KB}"
    saving_potential: "{percent}"
    has_srcset: true | false
    has_lazy_load: true | false
    has_dimensions: true | false
    has_alt_text: true | false

  checks:
    unoptimized_format:
      description: "PNG/JPG where WebP/AVIF would save >30%"
      severity: MEDIUM
      action: "Convert via *asset-optimize"

    missing_srcset:
      description: "Image without responsive variants"
      severity: MEDIUM
      action: "Generate srcset (375w, 768w, 1440w, 2880w)"

    missing_lazy_load:
      description: "Below-fold image without loading='lazy'"
      severity: MEDIUM
      action: "Add lazy loading attribute"

    missing_dimensions:
      description: "Image without explicit width/height (causes CLS)"
      severity: HIGH
      action: "Add width and height attributes"

    missing_alt_text:
      description: "Image without alt attribute"
      severity: HIGH
      action: "Add descriptive alt text (a11y requirement)"
```

### Phase 5: Usage Analysis

```yaml
usage_analysis:
  per_asset:
    file: "{path}"
    imported_by: ["{component1}", "{component2}"]
    rendered: true | false
    render_count: N
    context: "hero | card | background | icon | avatar | decorative"

  checks:
    orphan_assets:
      description: "Asset in public/ or src/assets/ but never imported/rendered"
      severity: LOW
      action: "Remove to reduce bundle/deploy size"

    single_use_large:
      description: "Asset > 100KB used only once"
      severity: MEDIUM
      action: "Consider lazy loading or on-demand fetch"

    duplicate_assets:
      description: "Same visual asset in multiple formats or copies"
      severity: LOW
      action: "Consolidate to single optimized version"
```

## Scoring

```yaml
health_score:
  formula: "100 - (critical * 15) - (high * 8) - (medium * 3) - (low * 1)"
  max_deductions:
    critical: 60  # max 4 critical issues = -60
    high: 40      # max 5 high issues = -40
    medium: 15    # max 5 medium issues = -15
    low: 5        # max 5 low issues = -5

  classification:
    90-100: "Healthy — assets well-managed and optimized"
    70-89: "Needs attention — some optimization or license gaps"
    50-69: "At risk — broken links, missing licenses, or heavy assets"
    0-49: "Critical — legal risk or significant broken assets"
```

## Output Format

```yaml
output:
  schema: "discovery-output-schema.yaml"
  sections:
    - summary: "N external assets, M issues, health score X/100"
    - inventory: "Table of all external assets with metadata"
    - broken_links: "List of unreachable URLs"
    - license_report: "License status per asset"
    - optimization_gaps: "Assets that need format/size optimization"
    - orphans: "Assets not used anywhere"
    - recommendations: "Prioritized action list"
```
