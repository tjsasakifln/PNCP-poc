# Project DNA Extraction

## When This Task Runs

- **Automatically** on first Apex activation in a project (before any agent routing)
- **Manually** via `*apex-scan --deep` for full re-extraction
- **Incrementally** when `*discover-design` or `*discover-components` results change
- Cache invalidation: re-runs if `package.json`, `tailwind.config.*`, or root CSS files change

## Purpose

Every Apex agent needs to know the project's conventions BEFORE making decisions. This task extracts the project's DNA once and makes it available to all 14 agents, eliminating:
- Agents suggesting patterns that conflict with existing conventions
- Hardcoded values when tokens exist
- Wrong naming conventions in generated code
- CSS approaches that conflict with the project's design system

## Execution Steps

### Step 1: Extract Stack DNA

```
READ package.json for:
  - Framework: next, react, vite, expo
  - UI library: react, react-native
  - Styling: tailwindcss, styled-components, emotion, css-modules
  - Animation: framer-motion, react-spring, gsap
  - State: zustand, jotai, redux, recoil
  - Forms: react-hook-form, formik
  - Testing: vitest, jest, playwright
  - Icons: lucide-react, heroicons, phosphor
  - 3D: three, @react-three/fiber

OUTPUT:
  stack:
    framework: "{detected}"
    ui: "{detected}"
    styling: "{detected}"
    animation: "{detected}"
    state: "{detected}"
    testing: "{detected}"
```

### Step 2: Extract Design Language DNA

```
SCAN CSS entry points (index.css, globals.css, tailwind config):
  - Color palette: CSS custom properties, Tailwind theme.extend.colors
  - Spacing scale: --spacing-*, Tailwind spacing
  - Typography: font families, size scale, weights used
  - Border radius: radius scale, default radius
  - Shadows: shadow scale, glass/glow effects
  - Z-index: z-index layers used
  - Design style: glass morphism, material, flat, neumorphism

OUTPUT:
  design_language:
    style: "{glass-morphism | material | flat | custom}"
    colors:
      primary: "{hex}"
      secondary: "{hex}"
      accent: "{hex}"
      surface: "{hex}"
      text: "{hex}"
    spacing_unit: "{4px | 8px | custom}"
    radius_default: "{px}"
    shadow_style: "{none | subtle | elevated | glass}"
    dark_mode: "{supported | not-supported}"
    theme_mechanism: "{css-vars | tailwind-dark | data-attribute}"
```

### Step 3: Extract Component Convention DNA

```
SCAN src/ or app/ for component patterns:
  - File organization: feature-based, type-based, atomic, flat
  - Naming: PascalCase files, kebab-case folders, index.ts barrel
  - Export style: named exports, default exports, barrel re-exports
  - Component pattern: functional only, class+functional, hooks-heavy
  - Props style: inline types, separate interface, Zod schema
  - State location: colocated, lifted, global store

OUTPUT:
  conventions:
    file_org: "{feature | type | atomic | flat}"
    component_naming: "{PascalCase | kebab-case}"
    folder_naming: "{PascalCase | kebab-case | camelCase}"
    export_style: "{named | default | barrel}"
    props_style: "{inline | interface | type-alias}"
    import_style: "{absolute | relative | alias}"
    import_alias: "{@/ | ~/ | none}"
```

### Step 4: Extract Routing DNA

```
SCAN for routing patterns:
  - Router: next/app, react-router, expo-router, tanstack-router
  - Structure: file-based, config-based
  - Layouts: nested layouts, shared headers
  - Auth: middleware, route guards, layout-level
  - API: route handlers, server actions, external API

OUTPUT:
  routing:
    router: "{detected}"
    structure: "{file-based | config-based}"
    auth_pattern: "{middleware | guard | layout}"
    api_pattern: "{route-handlers | server-actions | external}"
```

### Step 5: Extract Quality DNA

```
SCAN for quality patterns:
  - Linting: eslint config, custom rules
  - Formatting: prettier config
  - Type strictness: strict mode, noUncheckedIndexedAccess
  - Test patterns: Testing Library, snapshot, E2E
  - CI: GitHub Actions, pre-commit hooks

OUTPUT:
  quality:
    eslint_config: "{flat | legacy | custom}"
    ts_strict: "{true | false}"
    test_framework: "{vitest | jest}"
    test_style: "{testing-library | enzyme | snapshot}"
    ci: "{github-actions | none}"
```

### Step 6: Compile Project DNA

Merge all extractions into a single `project-dna.yaml`:

```yaml
# Auto-generated by Apex — do NOT edit manually
# Re-generated on: {timestamp}
# Trigger: {first-activation | manual-scan | cache-invalidation}

project_dna:
  extracted_at: "{ISO timestamp}"
  project_name: "{from package.json}"

  stack: { ... }         # Step 1
  design_language: { ... } # Step 2
  conventions: { ... }    # Step 3
  routing: { ... }        # Step 4
  quality: { ... }        # Step 5

  # Profile selection (feeds agent activation)
  apex_profile: "{full | web-next | web-spa | minimal}"

  # Agent-specific context (extracted for each agent)
  agent_context:
    css-eng:
      styling_lib: "{tailwind | css-modules | styled-components}"
      design_tokens: "{css-vars | tailwind-config | theme-object}"
      dark_mode: "{data-theme | class | media-query}"
    react-eng:
      react_version: "{19 | 18}"
      rsc_enabled: "{true | false}"
      state_management: "{zustand | jotai | context}"
    motion-eng:
      animation_lib: "{framer-motion | react-spring | css-only}"
      reduced_motion: "{implemented | missing}"
    a11y-eng:
      aria_usage: "{extensive | basic | missing}"
      focus_management: "{implemented | partial | missing}"
    perf-eng:
      bundler: "{vite | webpack | turbopack}"
      lazy_loading: "{implemented | partial | missing}"
      image_optimization: "{next-image | manual | none}"
```

### Step 7: Cache & Distribute

```yaml
cache:
  location: ".aios/apex-context/project-dna.yaml"
  ttl: "Until package.json, CSS entry, or tailwind config changes"
  invalidation:
    - "package.json modified"
    - "tailwind.config.* modified"
    - "globals.css / index.css modified"
    - "Manual: *apex-scan --deep"

distribution:
  mechanism: "All agents read project-dna.yaml before executing tasks"
  injection: "Emil injects relevant agent_context section into each agent's prompt"
  fallback: "If DNA not available, agents use their default heuristics"
```

## Quality Criteria

- DNA extraction completes in under 5 seconds (static analysis only)
- All 5 DNA dimensions extracted (stack, design, conventions, routing, quality)
- Agent-specific context generated for active profile agents
- Cache invalidation triggers correctly on config file changes

## Quality Gate

| Check | Pass Criteria |
|-------|---------------|
| Completeness | All 5 DNA dimensions extracted |
| Accuracy | Detected values match actual project |
| Performance | Extraction under 5 seconds |
| Cache | Invalidation works on config changes |

## Handoff

- project-dna.yaml feeds ALL Apex agents on every task execution
- Stack DNA feeds `*apex-scan` for profile selection
- Design language DNA feeds `*discover-design` as baseline
- Convention DNA feeds all code generation tasks for consistent output
