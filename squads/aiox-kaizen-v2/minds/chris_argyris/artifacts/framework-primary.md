# Framework Primário: Double-Loop Learning

**Type:** Primary Operating Framework
**Agent:** @kaizen-v2:memory-keeper
**Status:** Influência fundamental em organizational learning desde 1977

## Overview

Double-Loop Learning é o framework que distingue entre corrigir ações (single-loop) e questionar os pressupostos que geraram essas ações (double-loop). No kaizen-v2, o ciclo capture-reflect implementa ambos os loops: daily capture registra o que aconteceu (single-loop), reflect questiona por que e extrai patterns de aprendizado (double-loop).

## Framework 1: Single-Loop vs Double-Loop

```text
SINGLE-LOOP LEARNING:
  Detect error → Correct action → Same rules
  "O hook falhou → Corrigir o path → Tentar de novo"

DOUBLE-LOOP LEARNING:
  Detect error → Question assumptions → Change rules → New behavior
  "O hook falhou → Por que assumimos que o path seria relativo? →
   Regra: hooks sempre usam path absoluto → Atualizar todos os hooks"

KAIZEN-V2 IMPLEMENTAÇÃO:
  Single-loop: Daily capture registra o que aconteceu
  Double-loop: Reflect questiona pressupostos + extrai patterns reusáveis
```

## Framework 2: Teoria Esposada vs Teoria em Uso

```text
TEORIA ESPOSADA (o que dizemos):
  - "Nossos hooks são robustos"
  - "Seguimos boas práticas"
  - "Documentamos tudo"

TEORIA EM USO (o que os dailies mostram):
  - "Hooks falharam 3x esta semana por path errado"
  - "Ignoramos testes em 2 sessões"
  - "README está 2 versões desatualizado"

KAIZEN-V2 RESOLVE ISSO:
  - Dailies captura teoria em uso (ações reais)
  - Reflect compara com padrões esperados (teoria esposada)
  - Patterns revelam a lacuna → ação corretiva
```

## Framework 3: Rotinas Defensivas vs Aprendizado

```text
ROTINAS DEFENSIVAS (bloqueiam aprendizado):
  - "Não é minha responsabilidade" → Ninguém investiga a causa raiz
  - "Já tentamos isso" → Fecha possibilidades sem re-examinar
  - "É complexo demais" → Evita confrontar o problema real
  - "Funciona bem o suficiente" → Aceita debt sem questionar

COMO KAIZEN-V2 COMBATE:
  - Daily capture é não-julgamental: registra fatos, não culpas
  - Patterns decay naturalmente: "sempre fizemos assim" perde score
  - Reflect é sistemático: aplica 5 critérios, não opinião
  - Segurança psicológica: sistema é automático, sem exposição pessoal
```

## Diagnostic Questions (Double-Loop)

1. "O que esse pattern nos diz sobre nossos pressupostos?"
2. "Onde há lacuna entre o que dizemos e o que fazemos?"
3. "Que rotinas defensivas estão impedindo este aprendizado?"
4. "Estamos apenas corrigindo a ação ou mudando a regra?"
5. "Este pattern foi reinforced porque é útil ou porque é confortável?"
