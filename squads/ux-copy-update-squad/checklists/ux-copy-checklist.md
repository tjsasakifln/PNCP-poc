# UX Copy Update Checklist

Use this checklist for every copy update to ensure quality and consistency.

## Pre-Update

- [ ] User story or requirement documented
- [ ] Copy change approved by Product Owner
- [ ] New text reviewed by UX Designer (if applicable)
- [ ] Context understood (where text appears, audience, purpose)

## Text Quality

- [ ] Grammar and spelling correct (pt-BR)
- [ ] Tone matches brand voice (professional, helpful)
- [ ] User-centric language (focuses on benefits)
- [ ] Action-oriented phrasing (clear next steps)
- [ ] Concise without losing clarity
- [ ] No technical jargon (unless necessary)
- [ ] Abbreviations expanded on first use

## UX Validation

- [ ] Improves clarity vs. old copy
- [ ] Maintains or improves user experience
- [ ] Appropriate for context (welcome, loading, error, etc.)
- [ ] Consistent with other messaging
- [ ] UX score ≥70/100

## Accessibility

- [ ] Reading level appropriate (Grade 8-10)
- [ ] Screen reader compatible
- [ ] Sufficient context provided
- [ ] No reliance on visual cues alone
- [ ] WCAG 2.1 Level AA compliant

## Implementation

- [ ] All instances of old text found
- [ ] Backups created before changes
- [ ] Changes applied correctly
- [ ] No unintended side effects
- [ ] i18n keys updated (if applicable)

## Testing

- [ ] Desktop viewports tested
- [ ] Tablet viewports tested
- [ ] Mobile viewports tested
- [ ] Cross-browser compatibility verified
- [ ] No text overflow or truncation
- [ ] No layout shifts
- [ ] Keyboard navigation works
- [ ] Screen reader tested (NVDA/JAWS)

## Documentation

- [ ] Change documented in git commit
- [ ] Updated related documentation
- [ ] Informed stakeholders
- [ ] Added to changelog (if applicable)

## Final Review

- [ ] Product Owner approval
- [ ] UX Designer sign-off (if applicable)
- [ ] QA passed
- [ ] Ready to deploy

---

## Quick Check (Minimal)

For minor copy tweaks, use this abbreviated checklist:

- [ ] Text quality validated
- [ ] UX approved
- [ ] Accessibility checked
- [ ] UI tested (basic)
- [ ] Ready to deploy

---

## Notes

**Context-Specific Guidelines:**

- **Welcome Messages:** Focus on value and personalization
- **Loading States:** Be reassuring and specific
- **Error Messages:** Be clear, empathetic, actionable
- **CTAs (Buttons):** Be action-oriented and brief
- **Help Text:** Be instructive and contextual

**Tone Examples:**

✅ **Good:**
- "Estamos trabalhando nisso, só mais um instante!"
- "Encontre oportunidades mais adequadas para seu negócio"

❌ **Bad:**
- "Processando requisição... Aguarde."
- "Buscar licitações no banco de dados"
